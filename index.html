Notes and info

This converter was built without the use of a dictionary, parser, or grammar analyzer. As a result, it may be less accurate in cases where conversion depends on grammatical context. Please refer to the notes below for guidance on usage and limitations.

  • The particle は is romanized as "ha", even when pronounced "wa".
  • The particle へ should be entered as "he", not "e".
  • The particle を should be entered as "wo", not "o".
  • Japanese text typically does not include spaces. Since the tool cannot recognize word boundaries, the romanized output will also omit spaces (unless provided in the kana input).
  • Only valid Japanese syllables will be converted. For foreign words, the converter cannot automatically adapt them into appropriate Japanese phonetic equivalents. You’ll need to manually adjust the input to match valid syllables (e.g., use "Makudonarudo" for "McDonald's").

This tool supports both the Hepburn and Nihon-shiki romanization systems, with a few practical exceptions to reduce ambiguity:

  • Use "ei" for long e sounds and "ou" for long o sounds.
  • Other vowels are doubled: "aa", "ii", "uu".
  • This avoids the need for macrons (e.g., ā, ī, ū, ē, ō), which are less convenient to type.
  • じ / ジ is romanized as "ji", while ぢ / ヂ is romanized as "di".
  • ず / ズ is romanized as "zu", while づ / ヅ is romanized as "du".

The kana-to-romaji converter primarily uses the Hepburn system, with the same exceptions listed above to handle ambiguous cases consistently.

The converter supports numerals and basic punctuation.

Disclaimer: This project was created as a JavaScript practice exercise, and does not claim to be a fully accurate linguistic tool. While I’ve made every effort to cover edge cases and build a complete solution using only internal logic, the converter remains a rudimentary implementation. It does not use external linguistic data, and may produce imperfect results—especially in contexts that require deeper grammatical or semantic analysis.
